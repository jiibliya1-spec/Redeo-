مهمة إصلاح شاملة وعاجلة للمشروع — إصلاح فقط بدون حذف أي شيء

أنت تملك بالفعل:
- صلاحية GitHub
- صلاحية Supabase
- ملفات APK / Build
- Environment Variables
- قاعدة البيانات الحالية
- الكود الحالي للتطبيق

مهمتك:
إصلاح المشروع الحالي بشكل احترافي بدون حذف أي ميزة، بدون إعادة إنشاء المشروع، وبدون حذف أي كود أو صفحات أو بيانات.

قواعد مهمة جداً:
- لا تقم بإعادة بناء المشروع من الصفر
- لا تحذف أي صفحة أو Component أو Feature
- لا تكسر الـ Navigation
- لا تحذف أي جدول من Supabase
- لا تحذف الترجمات
- لا تغير تصميم التطبيق الحالي
- أصلح فقط المشاكل الموجودة
- حافظ على جميع البيانات الحالية

==================================================
1. إصلاح مشكلة رفع الوثائق
==================================================

المشكلة الحالية:
عند رفع أي وثيقة تظهر رسالة:
"فشل التحميل"

يجب إصلاح نظام الرفع بالكامل.

تحقق من:
- أسماء Buckets
- Storage Policies
- RLS Policies
- جلسة المستخدم Auth Session
- تحويل الصور إلى Blob
- توافق React Native / Expo
- امتدادات الملفات
- MIME Types
- Supabase Storage API

استعمل أسماء Buckets التالية حرفياً:
- cin-documents
- selfies
- driver-licenses
- vehicle-documents
- insurance-documents
- vehicle-photos
- avatars
- chat-images

المطلوب:
- رفع الملفات يشتغل على الهاتف الحقيقي
- حفظ روابط الملفات داخل قاعدة البيانات
- تحديث الحالة مباشرة بعد النجاح
- عدم إظهار نجاح وهمي
- إظهار Progress أثناء الرفع
- ظهور الملفات داخل Supabase Storage فعلياً

==================================================
2. إصلاح نظام التحقق Admin Verification
==================================================

المشكلة الحالية:
عند قبول أو رفض الوثائق من طرف الأدمن لا يتم تحديث المستخدم بشكل صحيح.

المطلوب:

عندما يرفع المستخدم الوثائق:
- الحالة تصبح pending

عندما يضغط الأدمن Approve:
- إرسال Notification فورية
- تحديث البروفايل مباشرة
- ظهور Verified Badge فوراً
- باقي المستخدمين يشاهدون الحساب موثق

عندما يضغط Reject:
- تصل رسالة رفض للمستخدم
- يستطيع إعادة رفع الوثائق

كل هذا بدون Logout أو Refresh.

==================================================
3. إصلاح نظام الإشعارات
==================================================

المطلوب:
- Realtime Notifications تعمل بالكامل
- حفظ الإشعارات داخل Supabase
- Toast يظهر فوراً
- عداد الإشعارات يتحدث مباشرة
- الإشعارات تبقى بعد إعادة تشغيل التطبيق
- Polling Fallback يعمل تلقائياً

==================================================
4. إصلاح نظام الترجمة i18n
==================================================

المشكلة الحالية:
ظهور مفاتيح مثل:
landing.hero_title
profile.logout
driver.title

بدل النص الحقيقي.

المطلوب:
- ترجمة جميع النصوص بالكامل
- عدم ظهور أي Translation Key
- العربية والفرنسية والإنجليزية تعمل بالكامل
- تغيير اللغة يحدث مباشرة
- جميع النصوص الجديدة مترجمة
- حذف أي مفاتيح مكررة أو ناقصة

==================================================
5. إصلاح نظام الصلاحيات والأدوار
==================================================

المشكلة الحالية:
حساب الأدمن يدخل أحياناً كأنه سائق عادي.

المطلوب:
- الأدمن يدخل دائماً إلى Dashboard الأدمن
- السائق يدخل واجهة السائق
- المسافر يدخل واجهة المسافر
- نفس الحساب يمكنه التحويل بين Driver و Passenger
- التحويل يتم فوراً بدون مشاكل
- حفظ Session بشكل صحيح

==================================================
6. إصلاح نظام المحادثات
==================================================

المطلوب:
- المستخدم لا يستطيع مراسلة نفسه
- السائق والمسافر يمكنهم التواصل
- الرسائل تعمل Realtime
- إرسال الصور يعمل
- حفظ الرسائل داخل Supabase
- إشعارات الرسائل تعمل

==================================================
7. إصلاح نظام البروفايل
==================================================

المطلوب:
- Verified Badge يظهر للجميع
- التقييمات تظهر بشكل صحيح
- صورة البروفايل تعمل
- تحديث البيانات يعمل
- التحويل بين Driver و Passenger يعمل
- Public Profile يعمل

==================================================
8. إصلاح الرحلات والبحث
==================================================

المطلوب:
- نشر الرحلات يعمل
- حفظ الرحلات داخل قاعدة البيانات
- البحث والفلاتر تعمل
- تحديث الرحلات Realtime
- نظام الحجز يعمل
- معلومات السائق والمسافر تظهر بشكل صحيح

==================================================
9. إصلاح Supabase بالكامل
==================================================

تحقق وأصلح:
- RLS Policies
- Storage Policies
- Realtime
- Indexes
- Foreign Keys
- Auth Triggers
- notifications table
- verifications table
- profile sync
- storage access

بدون حذف البيانات الحالية.

==================================================
10. الاختبار النهائي الإجباري
==================================================

يجب عليك اختبار كل شيء بنفسك فعلياً:

- إنشاء حساب جديد
- رفع جميع الوثائق
- تجربة قبول ورفض الوثائق
- اختبار الإشعارات الفورية
- اختبار Verified Badge
- اختبار الترجمة
- اختبار الرفع الحقيقي
- اختبار التطبيق على الهاتف

مهم جداً:
لا تتوقف عند كتابة الكود فقط.
قم بإصلاح المشاكل الحقيقية فعلياً حتى يصبح التطبيق جاهزاً للإنتاج بشكل احترافي.

إصلاح فقط.
بدون حذف.
بدون إعادة بناء المشروع من الصفر.
بدون كسر أي ميزة موجودة.
